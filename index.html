Why it exists

Maintenance tracking usually fragments fast.

Asset ownership changes. Equipment moves between people. Photos live in a phone. Parts research sits in another tab. WrenchLog puts those threads into one workflow, with a responsive UI and a backend that stays simple to deploy.

The repo is set up for development and image-style deployment, with the frontend and API sharing a single external port so the app behaves like one product instead of a pile of disconnected tools.

Operations

Docker Compose supports a development workflow and an image-only deployment path on port 5000.

Run modes

Two clear ways to launch the app.

Development mode

Use the compose override to mount source, keep Vite internal, and still reach the whole app through one external port.

docker compose up --build

Image-only deployment

Build the frontend into static assets, copy them into the backend image, and serve everything from Flask.

docker compose -f docker-compose.yml up --build

Highlights

Built-in details that make the repo more useful than a CRUD demo.

  • Project photos support png, jpg, jpeg, gif, webp, and bmp uploads.
  • Upload size is capped at 16 MB per request for predictable handling.
  • Vehicle detail pages can surface live read-only parts listings through RockAuto lookup.
  • The frontend can override its API base URL with VITE_API_BASE_URL when needed.
